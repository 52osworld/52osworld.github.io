文档概述 ：我希望知道的内容

在进一步阅读之前，请确保您已看过了“ 软件概述”页面，该页面介绍了此处引用的许多术语的定义。另外，要知道现在这里的一些信息对你意义不大，但是当你开始开发时，这些都是你想要迫切知道的信息。这些是使用YoctoProject®的最有名的工作方法，文档会定期更新。

---------

使用Yocto项目相当简单， 除非出现某些问题 。如果不了解构建过程的工作原理，您会发现自己在试图排除“黑匣子”的故障。以下是一些新用户希望他们在开始使用Yocto Project进行第一次构建之前了解的一些内容。如果有其他建议随时与我们联系。

1.使用Git，而不是tarball下载 。如果你使用git，由于git的工作方式，软件会自动更新bug更新。如果你下载了压缩包，你需要自己更新压缩包的内容。

2.了解Layer索引
所有Layer都可以在Layer索引中找到。可以在Yocto Project Compatible索引中找到适用于Yocto Project兼容状态（结构连续性保证和测试）的Layer。 一般首先检查兼容Layer索引，如果没有找到必要的Layer检查通用Layer索引。Layer索引是来自Open Embedded Project的原始工件。通用Layer索引并没有Yocto Project兼容Layer索引中提供的策划和测试，但后者的条目较少。当你在Layer索引中搜索时要知道，并非所有Layer都具有相同等级的成熟度，验证测试或可用性。搜索显示结果也不会有优先级排列。没有简单的方法来帮助您选择适应您的需求的最佳Layer。因此，往往需要反复试验，检查邮件列表，或通过协作室与其他开发人员合作，以此来帮助您做出正确的选择。

3.尽可能使用芯片供应商现有的BSP层
英特尔，TI，恩智浦和其他公司都有关于他们芯片应该使用哪种BSP层的信息。这些Layer具有诸如“meta-intel”或“meta-ti”等名称。尽量不要从头开始构建Layer。如果您有自定义芯片，请将其中一个Layer用作指导或模板，并熟悉“ Yocto项目电路板支持包（BSP）开发指南” 。

4.不要把所有东西都放到一个图层中
使用不同的Layer来在逻辑上分离构建中的信息。例如，您可以拥有BSP层，GUI层，发行版配置，中间件或应用程序（例如“meta-filesystems”，“meta-python”，“meta-intel”等）。将整个构建放在一个Layer中会限制未来的定制和重用。反之，将信息隔离为多个层次有助于简化未来的定制和重用。

5. 永远不要修改POKY Layer。永远不要。永远！当你更新到下一个release版本时，会导致你之前所有的工作没有意义。所有的一切！

6.不要被文档搜索结果愚弄
Yocto项目文档一直在更新。不幸的是，当您使用Google搜索Yocto项目的概念或术语时，Google会持续搜索并检索旧版本的Yocto Project手册。例如，使用Google搜索特定主题可能会搜索到几个版本之前的Yocto Project手册。为确保您使用的是最新的Yocto项目文档，请使用Yocto Project文档页面为您的软件版本找到正确的文档。如果您使用“文档概述”页面顶部的搜索栏，但该搜索不是最佳搜索栏，则会将您指向可找到该搜索字符串的文档。这种搜索通常会指出给定术语或概念在哪里。

许多开发人员通过搜索“Yocto Project Complete Documentation Set”来查看完整的Yocto Project手册集的概念或术语。本手册是Yocto项目手册的核心组合。因此，在本手册中使用Ctrl-F进行简单的字符串搜索会为所需的术语或概念生成所有“匹配”。一旦找到您感兴趣的区域，如果需要，您可以显示实际的手册。

7.了解构建系统如何工作的基本概念：工作流程
了解Yocto项目工作流程非常重要，因为它可以帮助您找出故障发生的位置以及构建是如何中断的。工作流程分解成以下步骤：

  • 1）Fetch - 获取源代码，
  • 2）Extract - 解压源代码
  • 3）Patch - 应用补丁修复错误和新功能
  • 4）Configure - 设置您的环境
  • 5）Build - 编译和链接
  • 6）Install - 将文件复制到目标目录
  • 7）Package - 打包用于安装的软件包文件。

在“Fetch”期间，可能无法找到代码。在“Extract”期间，可能会出现无效的zip或类似内容。换句话说，工作流程的特定部分的功能可以让您了解可能出现的问题。

8.要知道你可以生成一个依赖图并学习如何去做
依赖关系图显示配方，任务和目标之间的依赖关系。您可以在BitBake中使用“-g”选项来生成此图。当你开始构建并且构建中断时，你可以找到需要的包，或者知道为什么构建系统已经包含它们。依赖关系图可以阐明这种困惑。您可以在BitBake User Manual的“ Generating Dependency Graphs”章节了解更多关于依赖关系图的信息以及如何生成它们。

9.以下是在tmp / work中解码“magic”文件夹名称的方法
构建系统进行提取，解包，预处理和构建操作。如果出现问题，构建系统会直接向您报告临时（build / tmp）文件和包所在的文件夹的路径，这些文件和包由构建系统生成。有关此过程的详细示例，请参阅示例 。不幸的是，这个例子是Yocto项目的早期版本。

在执行构建时，可以使用“-u”BitBake命令行选项将用户界面查看器指定到依赖关系图中（例如knotty，ncurses或taskexp），以帮助您更好地理解构建依赖关系。

10.您可以构建的不仅仅是镜像
您可以为特定的软件包（包括devshell）甚至单个配方构建和运行特定的任务。当开发人员首次开始使用Yocto项目时，Yocto项目快速入门中的说明指示你如何创建镜像，然后运行或刷入该镜像。无论如何，您实际上可以只构建一个配方。因此，如果某些依赖项或配方不起作用，则可以仅说“bitbake foo”，其中“foo”是特定配方的名称。随着您更深入地使用Yocto，如果构建失败，这可以确保获取本身能够按照需要工作。以下是一些有价值的链接：“Using a Development Shell” ，了解如何使用devshell构建和运行特定任务。 此外， SDK手册还介绍了如何构建特定配方。

11.一个模糊的定义：Package vs Recipe
配方包含构建系统用于创建包的说明。配方和包是分别是前端和构建过程的结果。

如前所述，构建系统会获取配方并根据配方的说明创建包。生成的包与配方构建的一件事有关，但是是构建的不同部分（包）（即主包，doc包，调试符号包，分立的utilities包等等）。构建系统将软件包分割出来，这样您就可以安装不想或不需要的软件包，这对嵌入式和物联网开发小型设备非常有利。

12.您将想要了解并知道rootfs中打包的内容

13. 创建您自己的镜像配方
有许多方法可以创建您自己的镜像配方。我们建议您创建自己的镜像配方，而不是添加现有配方。编写镜像配方是很简单和容易的。再次，不要尝试附加到现有的镜像配方。创建你自己的配方，并从一开始就这样做。

14。最后，这里列出了您作为系统开发人员需要的基本技能。您必须能够：

  • 处理公司代理
  • 将一个包添加到镜像
  • 了解配方和包装之间的区别
  • 自行构建一个包，为什么这很有用
  • 找出配方创建了哪些软件包
  • 找出包中的文件
  • 找出镜像中的文件
  • 添加一个SSH服务器到镜像（启用文件传输到目标）
  • 知道如何剖析配方
  • 知道如何创建和使用图层
  • 找到配方（layers.openembedded.org）
  • 了解机器和发行版设置之间的区别
  • 为您的硬件找到并使用正确的BSP（机器）
  • 查找特定发行版的示例并知道如何设置它们
  • 了解任务管道并执行单个任务
  • 了解devtool以及它如何简化您的工作流程
  • 共享下载和共享状态缓存提高构建速度
  • 生成并理解依赖关系图
  • 生成并理解bitbake环境
  • 为应用程序开发构建可扩展的SDK

15根据您对Yocto项目的主要兴趣，您可以考虑以下任何阅读材料：

查看Yocto项目开发任务手册 ：本手册包含分组的程序信息，以帮助您设置，处理图层，自定义镜像，编写新配方，使用库和使用QEMU。这些信息是基于任务的，涵盖了Yocto项目的广度。

查看Yocto项目应用程序开发和可扩展软件开发工具包（eSDK）手册 ：本手册描述了如何使用标准SDK和可扩展SDK，这些SDK主要用于应用程序开发。本手册还提供了使用流行的Eclipse™开发环境并使用devtool的示例工作流程。有关更多信息，请参阅“使用Eclipse™的工作流程”和“在SDK工作流程中使用devtool”部分。

了解内核开发 ：如果您想了解如何使用内核并了解Yocto Linux内核，请参阅Yocto Project Linux内核开发手册。本手册提供了有关如何给内核打补丁，修改内核配方和配置内核的信息。

了解板级支持包（BSP） ：如果您想了解BSP，请参阅Yocto项目板级支持包（BSP）开发人员指南。本手册还提供了示例BSP创建工作流程。请参阅“开发板级支持包（BSP）”部分。

了解烤Toaster ：Toaster是Yocto项目的OpenEmbedded构建系统的Web界面。如果您有兴趣使用此类界面创建镜像，请参阅Toaster用户手册。

提供Yocto项目参考手册 ：与Yocto Project手册其余部分不同，本手册提供了参考材料，而非过程步骤。您可以获取构建细节，仔细了解Yocto项目开发环境的各个部分如何协同工作，了解各种技术细节，有关迁移到较新的Yocto Project release 版本的指导，目录结构，类和任务的参考资料。Yocto项目参考手册还包含Yocto项目中使用的相当全面的变量词汇表。
